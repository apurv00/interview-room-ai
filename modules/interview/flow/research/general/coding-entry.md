# General / Any Role — Coding Interview — Entry Level (0-2 years)

## Topic Sequence (typical order)
1. **Array and string manipulation** (two pointers, sliding window)
2. **Hash maps and sets** for lookup optimization
3. **Stack and queue** applications
4. **Basic recursion** and tree traversal
5. **Sorting and searching** (binary search)
6. **Basic graph traversal** (BFS/DFS)
7. **Big O complexity** analysis
8. **Edge case handling**

## Phase Structure
- Online assessment (2-3 easy/medium, 60-90 min) → Phone screen (1 medium, 45 min) → Onsite (2-3 rounds)
- Easy → Medium LeetCode difficulty
- Pattern recognition: sliding window, two pointers, hashing, recursion
- Must communicate thought process throughout

## What Makes This Level Unique
- Success from **recognizing patterns** and applying right technique
- Problem-solving approach valued over optimal solution
- Minimum prep: ~150 problems across core patterns
- Must handle basic edge cases (empty input, single element, duplicates)
- Demonstrating systematic breakdown matters more than encyclopedic knowledge

## Anti-Patterns
- Going silent while coding
- Jumping to code without confirming requirements
- Ignoring edge cases
- Vague complexity analysis
- Memorizing solutions without understanding patterns
