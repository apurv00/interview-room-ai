# General / Any Role — Coding Interview — Senior Level (7+ years)

## Topic Sequence (typical order)
1. **Complex algorithm design** (advanced DP, graph, optimization)
2. **System-level code architecture** (module boundaries, interfaces, extensibility)
3. **Scalable code patterns** (concurrency, distributed algorithms)
4. **Technical leadership** demonstration through code
5. **API contract design** and backward compatibility
6. **Performance profiling** and production-grade optimization
7. **Error handling, resilience,** and fault tolerance in code
8. **Code review and mentoring** perspective

## Phase Structure
- Multiple rounds: 2-3 coding + system design + architecture + behavioral leadership
- Hard LeetCode + real-world system problems
- May include "design and implement" hybrid (architecture expressed in code)
- Some companies (Meta 2025+) piloting AI-enabled coding environments

## What Makes This Level Unique
- Coding becomes vehicle to demonstrate **architectural thinking**
- Tests designing scalable, secure systems and guiding other engineers
- System design separates senior from mid — choose trade-offs, justify, highlight techniques
- Must demonstrate **mentoring capability** in how you approach problems
- Product Architecture rounds: API design, client-server interaction, product evolution
- Bar: not just "correct and optimal" but **"production-ready and maintainable by a team"**

## Anti-Patterns
- Solving problem but ignoring code organization and extensibility
- Not considering production environment (error handling, logging, monitoring)
- Inability to discuss design trade-offs at code level
- Clever but unreadable code
- Not demonstrating leadership qualities
- Focusing solely on algorithmic complexity while ignoring real-world constraints
