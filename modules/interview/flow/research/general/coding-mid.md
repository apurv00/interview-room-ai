# General / Any Role — Coding Interview — Mid Level (3-6 years)

## Topic Sequence (typical order)
1. **Medium-difficulty algorithms** (DP, graph algorithms, greedy)
2. **Code quality** and clean code (DRY, modular helpers)
3. **Optimization** from brute force to optimal
4. **Complex data structures** (heaps, tries, union-find)
5. **Concurrency and threading** awareness
6. **Testing and debugging** methodology
7. **API and interface design** in code
8. **Time/space complexity trade-off** articulation

## Phase Structure
- Phone screen (1-2 medium, 45-60 min) → Onsite (2-3 coding + system design + behavioral)
- Medium to Medium-Hard LeetCode
- Evaluated: correctness, code quality, optimization, communication
- Clean modular code with helpers and idiomatic language usage

## What Makes This Level Unique
- Evaluation shifts from "can they solve it" to **"how well do they solve it"**
- Code quality is first-class: DRY, proper abstractions, readable names, modular helpers
- Start with correct brute force then iteratively optimize, articulating trade-offs
- Language mastery matters — use idioms, not just syntax
- Live debugging tasks may supplement algorithm problems

## Anti-Patterns
- Working but messy, unmaintainable code
- Cannot optimize beyond brute force
- Cannot articulate why one approach beats another
- Skipping test cases and edge case discussion
- Not using language features idiomatically
- Over-optimizing before having correct solution
