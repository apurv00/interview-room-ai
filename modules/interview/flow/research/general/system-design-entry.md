# General / Any Role — System Design Interview — Entry Level (0-2 years)

## Topic Sequence (typical order)
1. **Basic client-server-database** architecture
2. **API design fundamentals** (REST basics)
3. **Simple data modeling**
4. **Introduction to load balancing** and caching concepts
5. **Basic scalability reasoning** (vertical vs. horizontal)
6. **Understanding CAP theorem** at high level
7. **Simple system questions** (URL shortener, rate limiter, chat app)

## Phase Structure
- Interviewer-driven (guides and prompts, candidate follows)
- 30-45 min, single system, breadth over depth
- Focus: "Do you understand how basic components interact?"
- Not expected to provide production-ready designs

## What Makes This Level Unique
- Know basics but not implementation details (load balancer is necessary but not NGINX vs. AWS ALB)
- Focus on **potential and foundational reasoning**
- Most common: URL shortener, rate limiter, real-time chat, basic API design
- Can excel with solid fundamentals and first-principles reasoning

## Anti-Patterns
- Diving in without clarifying requirements
- Showing off with advanced patterns without understanding basics
- Not explaining why components exist (just drawing boxes)
- Overcomplicating simple designs
- Forgetting data flow between components
