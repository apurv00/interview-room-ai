# Frontend Engineer — Technical Interview — Entry Level (0-2 years)

## Topic Sequence (typical order)
1. **HTML Semantics & Structure** — Semantic elements (`<article>`, `<section>`, `<nav>`, `<main>`), forms, `<meta>` tags, document structure
2. **CSS Fundamentals** — Box model, specificity/cascade, positioning (static/relative/absolute/fixed/sticky), Flexbox, Grid, responsive media queries, units (rem/em/vw/vh)
3. **CSS Layout Challenges** — "Center this element vertically and horizontally" (multiple approaches); responsive grid; holy grail layout; mobile-first
4. **JavaScript Core** — Scope, hoisting, closures, `this`, event loop basics, type coercion, `var`/`let`/`const`, arrow vs regular functions
5. **DOM Manipulation** — `querySelector`, event listeners, event delegation/bubbling, creating/modifying/removing elements, traversal
6. **Basic React/Framework Concepts** — JSX, props vs. state, `useState`/`useEffect`, component lifecycle, Virtual DOM concept
7. **Responsive Design & Cross-Browser** — Media queries, breakpoints, mobile-first, `srcset`, viewport meta, cross-browser awareness
8. **Basic Accessibility** — `alt` attributes, form labels, heading hierarchy, color contrast, keyboard nav awareness

## Phase Structure
- **Fundamentals quiz (10-15 min):** Quick-fire on HTML/CSS/JS fundamentals — short answers (box model, specificity, scope)
- **Coding exercise (20-25 min):** Build small UI component (counter, todo, toggle) or JS utility (flatten array, implement `map`)
- **Framework basics (5-10 min):** Chosen framework questions — how rendering works, state vs. props, basic hooks
- **Wrap-up (5 min):** "How would you make this accessible?" / "What would you do differently?"

## What Makes This Level Unique
- **HTML/CSS carry equal weight to JavaScript** — unlike mid/senior where JS dominates
- Expect solving problems **with vanilla HTML/CSS/JS first** before frameworks
- Less emphasis on algorithms, more on **domain knowledge applied to real-world scenarios**
- Coding challenges involve **building actual UI components** (counter, card, form), not LeetCode puzzles
- Expect knowledge of **one framework** (usually React) at surface level
- **CSS layout ability** (Flexbox/Grid) is a strong differentiator — many bootcamp grads struggle with CSS
- Virtual DOM questions test **understanding how their framework works**, not just how to use it

## Anti-Patterns (do NOT ask at this level)
- Heavy algorithm/data structure questions (save for backend)
- State management libraries (Redux, Zustand) — too advanced
- Build tools, webpack configuration, CI/CD pipelines
- Obscure browser APIs or JS trivia (`typeof null === 'object'`)
- SSR, SSG, or rendering strategies
- TypeScript generics or advanced type system

## Probe Patterns
- After CSS solution: "Can you solve this a different way?" (Flexbox vs. Grid vs. absolute positioning)
- After JS answer: "What if input is null/undefined/empty array?" (edge case thinking)
- After React answer: "What does the framework do with your JSX under the hood?" (understanding vs. memorization)
- DOM manipulation: "Why event delegation instead of listeners on each element?" (performance awareness)
