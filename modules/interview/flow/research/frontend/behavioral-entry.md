# Frontend Engineer — Behavioral Interview — Entry Level (0-2 years)

## Topic Sequence (typical order)
1. **Motivation for Frontend** — Why frontend over backend/fullstack; what sparked interest in UI development
2. **Learning a New Framework/Tool** — Picking up React, Vue, or CSS framework for the first time; approach to the learning curve
3. **First Responsive Design Challenge** — Building first mobile-responsive layout; breakpoints, flexbox/grid decisions
4. **Working with a Designer** — Disagreeing on implementation of a design mockup; handling Figma handoff; pixel-perfect vs. pragmatic tradeoffs
5. **CSS Battle / Cross-Browser Bug** — Layout that broke in Safari or IE; debugging CSS visible only on specific viewports
6. **Shipping Under Deadline** — Cutting scope on a UI feature; deciding what visual polish to sacrifice
7. **Receiving Feedback on UI Work** — Getting subjective design critique ("it doesn't feel right"); handling non-technical feedback on visual output
8. **Team Collaboration on a Frontend Project** — Shared codebase; merge conflicts in CSS/component files; pair programming on UI

## Phase Structure
- **Warm-up (5 min):** "What got you interested in frontend?" / "Tell me about a UI you built that you're proud of"
- **Core behavioral stories (20-25 min):** 3-4 STAR questions on collaboration, learning, and frontend-specific challenges
- **Wrap-up (5 min):** "What kind of frontend challenges excite you most?"

## What Makes This Level Unique
- Stories come from **bootcamps, personal projects, internships** — not production systems at scale
- Interviewers look for **learning velocity and self-awareness**, not depth of experience
- Emphasis on handling **subjective feedback on visual work** (unique to frontend — backend juniors don't face "it doesn't look right")
- Expect candidates to discuss **specific UI frameworks/tools** they used (React, Tailwind, etc.)
- Questions probe **user-facing impact** even for small projects — "did anyone use it? what did they think?"

## Anti-Patterns (do NOT ask at this level)
- Leading teams, mentoring, or architecture decisions
- CSS specificity rules or JS closures during behavioral rounds
- Hypothetical "what would you do if..." instead of "tell me about a time..."
- Production-scale war stories
- Generic "tell me about a conflict" without frontend context

## Probe Patterns
- Learning React: "What was the hardest concept? How did you get unstuck?" (independent learning vs. tutorial dependency)
- Working with designer: "Walk me through the specific disagreement. What was the designer's reasoning?" (seeing both sides)
- CSS bug: "How did you debug it? What tools?" (DevTools, browser testing — practical skill)
