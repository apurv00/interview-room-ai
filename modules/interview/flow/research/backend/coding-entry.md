# Backend Engineer — Coding Interview — Entry Level (0-2 years)

## Problem Types
- **Arrays & Strings:** Two-pointer, sliding window, basic manipulation (most common)
- **Hash Tables:** Frequency counting, lookup optimization
- **Linked Lists:** Reversal, cycle detection
- **Stacks & Queues:** Balanced brackets, basic BFS
- **Trees:** Basic traversal (in-order, pre-order, BFS), BST search
- **Sorting & Searching:** Binary search, merge sort concepts
- **Recursion:** Simple recursive problems, backtracking basics
- **Backend-practical (growing trend):** REST API CRUD, database queries, simple data transformations

## Difficulty Level
- **Easy: 50-60% | Medium: 35-45% | Hard: 0-5%**
- Most interviews stay Easy-to-Medium; bar is "can you solve a medium with hints?"
- Interview prep guidance emphasizes 60-80 Easy problems first, then 40-60 Medium

## Phase Structure (30-45 minutes)
1. **Problem Intro (2 min):** Interviewer presents problem with example
2. **Clarification (3-5 min):** Ask about input constraints, edge cases, output format
3. **Approach Discussion (5 min):** Describe brute force, then optimize; state time/space complexity
4. **Coding (15-20 min):** Implement solution; interviewer may give hints
5. **Testing (5 min):** Walk through 1-2 examples, identify edge cases
6. **Optimize/Follow-up (5 min):** Can you do better? What if input is very large?

## What Makes This Level Unique
- Interviewers ask: **"Can they learn?"** — evaluating potential, not expertise
- **Hints are expected and acceptable;** asking for help is not penalized
- **Working brute-force solution is often sufficient to pass**
- Practical skills increasingly valued: 66% of developers prefer practical coding challenges; companies test REST API development, database queries over abstract algorithm puzzles
- **60-90 minute take-home challenges** (build a small API) are common alternatives to live coding
- **AI/LLM coding tools policy** is evolving (2025-2026) — some companies allow, others prohibit

## Evaluation Criteria
1. **Problem Solving:** Can you break down the problem logically? Multiple approaches considered?
2. **Communication:** Think aloud, ask clarifying questions, explain reasoning
3. **Technical Competency:** Correct implementation, basic language constructs
4. **Testing:** Trace through test cases, catch basic bugs
5. **Fundamentals:** DRY, CRUD, basic data structure selection

## Anti-Patterns (do NOT expect at this level)
- Hard dynamic programming or graph theory problems
- System design questions
- Flawless, production-quality code with full error handling
- Distributed systems concepts
- Being penalized for brute force if you communicate the trade-off

## Common Backend-Specific Problems
- Build a simple REST endpoint that processes and returns JSON
- Two Sum / Three Sum (hash map)
- Valid Parentheses (stack)
- Merge Sorted Arrays
- Reverse a Linked List
- Binary Search on sorted data
- Simple database query optimization (SQL joins, indexing basics)
- FizzBuzz-style data transformation with backend context (parse logs, filter records)
